# Lion Man 6조 Ground Rule

## 스프린트(2 일)

#### 스프린트 계획(2-3 시간)

스프린트 목표 설정, 백로그에서 스프린트에 수행 할 작업 정의

**스프린트 계획**
- 스프린트 주기는 **2일** 로 정합니다.
- 한 스프린트가 끝날 때마다 다음 스프린트의 계획을 세움

#### 스프린트 실행 & 일일 스크럼(15 분)

**AM 09:30 - AM 9:45**

- 스프린트 진행 동안에 `Projects` 참고하여 각자 무엇을 해야할지 보고 개발
- 일일 스크럼은 자기 시간에 아주 짧고 명확히 공유(1-2 분)
- 공유할 때는 `Projects` 에서 `Ready` 와 `In Progress`

#### 스프린트 리뷰(1 시간)

**PM 4:00 - PM 4:30**

스프린트가 끝난 후에는 자기가 스프린트에서 진행했던 결과물 혹은 진행되고 있는 상황을 이해관계자에게 공유 및 피드백을 하여야 한다. 피드백이 끝난 후에는 회고를 가진다.

#### 스프린트 회고(20 분)

**PM 4:30 - PM 4:50**

팀이 프로세스와 협업 방식을 되돌아보고 다음 스프린트에서 개선할 점을 찾아야 한다. 개선점이 없을 때는 후에 진행할 것들에 대한 토의로 계획으로 넘어간다.

## 짝 프로그래밍

무조건 두 명이서 짝지어서 개발, 짝이 없을 경우 3명이서 개발

- 재훈, 도은
- 성훈, 민혁
